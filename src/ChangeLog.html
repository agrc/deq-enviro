0.8.0 Aug 6, 2014

  • Updated legend labels for the "Indian Country and Tribal Lands" reference layer (#3 & #121).
  • Added related tables to identify pane (#116). Still need to handle nested relationships (#127).
  • Added a help title popup to the map layers button (#122).
  • Adjusted transparency of result polygons down to 75% (#124).
  • Fixed a bug that was causing strange zooming when searching by address or buffered shapes (#126).
  • Fixed a bug that was causing field aliases with parentheses to not show up in the identify attributes (#135).
  • Fixed a misspelling in the "Hydrologic Units" reference layer label (#136).
  • Fixed a bug that was causing the "There was an error with the search service!" error with certain layers (#137).
  • Fixed a variety of misspellings of field names in the query layers doc (eg. (#140). Added functionality to the nightly update script to validate field names (#141). I'm sure that there are more that need to be fixed.

0.7.0 Jul 28, 2014

  • Made improvements to the error log that is emailed by the nightly script if there are data errors (#3 & #107).
  • Fixed a bug that was preventing an appropriate error message from being displayed if there was an error returned from the search service (#3 & #109).
  • Created the identify pane and associated ellipsis (...) button on the grid. Also wired identify pane to show when clicking on a result feature on the map (#3 & #110).
    • "Attributes" tab shows additional attribute fields from query layers.
    • "Links" tab shows links to external pages with the five main fields passed as url query parameters.
    • "Zoom to feature" and "Back to results" links also working.
    • Related Records is not implemented yet (#3 & #116).
  • Made links in the "Links" tab in the identify pane only show up if there is a value in the associated column in the query layers spreadsheet (#3 & #22).
  • The map extent is now preserved when the results grid is shown or hidden. Also added buttons to allow you to show or hide the grid (#3 & #111).
  • Fixed a layout bug that was causing the "No results" message to be cut off in the grid (#3 & #112).
  • Fixed a bug that was preventing the search graphics from clearing when switching search types (#3 & #114).
  • Fixed a bug that was preventing the search by name function to find all features (#3 & #115).
  • Cleaned up and simplified the grid colors and layout.

0.6.0 Jul 18, 2014

  • Major components of nightly script built. Still some data issues left to work through (#3 & #70).
  • The "Land Ownership" and "Indian Country and Tribal" reference layers now have popup legends (#25).
  • The result graphics now show up on top of the yellow search graphic (#97). In addition, result points will always be drawn on top of result polygons.
  • Added "water" to wizard text as per Mark Jenson (#102).
  • Added support for generic polygon symbology (#105). Custom symbology is not supported in the app currently but there are plans to look at it in the future.

0.5.0 Jun 20, 2014

  • Selected query layers are now persisted between sessions (#48).
  • Polygon searchs now honor the buffer value (#61).
  • Any existing yellow search graphics are cleared when you switch shape type (#62).
  • "Clear Parameters" button now clears all search parameters including selected query layers, all search fields and any existing search geometry (#64).
  • When you switch search types all search parameters are cleared (#65).
  • Search results now show up as points on the map (#72).
  • There is now a better error message displayed when you try to search without selecting a search type (#78).
  • Fixed a bug that occurred when you selected multiple query layers, then searched, then unselected a query layer and tried to search again. (#85).
  • Added the total features found for each query layer in the grid header (#86).
  • The entire grid header row that expands the results is now clickable instead of having to click on the tiny triangle (#88).
  • The grid is now cleared in-between searches (#90).
  • Fixed a bug that showed the grid and loader even if there was an error with the search (#96).

0.4.0 May 27, 2014

  • Search service results now displayed in popup grid (#71).
  • Fixed bug in county search with Beaver county (#63).
  • Fixed layout bug in Wizard widget (#60).
  • Back end search service (#40).
  • Only query layers that have actual data in the database are shown.

0.3.0 Apr 24, 2014

  • Wizard widget (#20).
    • Closed state is persisted between sessions.
    • "Back" and "Next" buttons automatically expand relevant panes.
  • Address search (#36).
  • City, County and Statewide searches (#37).
  • Name and ID searches (#38).
  • Shape search (#39).
  • Updated Indian Country and Tribal layer to be symbolized with two colors (#47).

0.2.0 Apr 10, 2014

  • Nightly script was created to scrape the Query Layers spreadsheet for config data for the app. Any changes made to the spreadsheet will show up in the app the next morning (#23).
  • Query Layers are grouped by heading (#18).
  • Define Search Location pane layout fleshed out including drop down, stack container and buttons (#19).
  • The disclaimer dialog was replaced with an external link from the Query Layers spreadsheet "Other Links" tab. This data is also scraped nightly so that the URL can be maintained by DEQ (#21).
  • The Township/Range/Section checkbox and label are now disabled at the smaller scales when the layer would not be visible on the map. A popup is also displayed to help the user know what's going on. The popup goes away and the controls are enabled as the user zooms into scales at which the layer would be visible (#24).
  • Query layer features are no longer displayed when you check them in the "Select Query Layers" pane. The plan is to only show found features on the map after a search is performed (#26).

0.1.0 Mar 28, 2014

  • Initial app layout framework and build system wired up.
  • Added Disclaimer link and associated modal dialog (#11).
  • Fleshed out initial draft of QueryLayer widget. (#10).
    • Toggle features on the map with checkbox.
    • Shows layer description and links to meta data page.
