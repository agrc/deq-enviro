Utah Environmental Interactive Map Change Log

Back to the app

0.5.0 Jun 20, 2014

  • Selected query layers are now persisted between sessions (#48).
  • Polygon searchs now honor the buffer value (#61).
  • Any existing yellow search graphics are cleared when you switch shape type (#62).
  • "Clear Parameters" button now clears all search parameters including selected query layers, all search fields and any existing search geometry (#64).
  • When you switch search types all search parameters are cleared (#65).
  • Search results now show up as points on the map (#72).
  • There is now a better error message displayed when you try to search without selecting a search type (#78).
  • Fixed a bug that occurred when you selected multiple query layers, then searched, then unselected a query layer and tried to search again. (#85).
  • Added the total features found for each query layer in the grid header (#86).
  • The entire grid header row that expands the results is now clickable instead of having to click on the tiny triangle (#88).
  • The grid is now cleared in-between searches (#90).
  • Fixed a bug that showed the grid and loader even if there was an error with the search (#96).

0.4.0 May 27, 2014

  • Search service results now displayed in popup grid (#71).
  • Fixed bug in county search with Beaver county (#63).
  • Fixed layout bug in Wizard widget (#60).
  • Back end search service (#40).
  • Only query layers that have actual data in the database are shown.

0.3.0 Apr 24, 2014

  • Wizard widget (#20).
    • Closed state is persisted between sessions.
    • "Back" and "Next" buttons automatically expand relevant panes.
  • Address search (#36).
  • City, County and Statewide searches (#37).
  • Name and ID searches (#38).
  • Shape search (#39).
  • Updated Indian Country and Tribal layer to be symbolized with two colors (#47).

0.2.0 Apr 10, 2014

  • Nightly script was created to scrape the Query Layers spreadsheet for config data for the app. Any changes made to the spreadsheet will show up in the app the next morning (#23).
  • Query Layers are grouped by heading (#18).
  • Define Search Location pane layout fleshed out including drop down, stack container and buttons (#19).
  • The disclaimer dialog was replaced with an external link from the Query Layers spreadsheet "Other Links" tab. This data is also scraped nightly so that the URL can be maintained by DEQ (#21).
  • The Township/Range/Section checkbox and label are now disabled at the smaller scales when the layer would not be visible on the map. A popup is also displayed to help the user know what's going on. The popup goes away and the controls are enabled as the user zooms into scales at which the layer would be visible (#24).
  • Query layer features are no longer displayed when you check them in the "Select Query Layers" pane. The plan is to only show found features on the map after a search is performed (#26).

0.1.0 Mar 28, 2014

  • Initial app layout framework and build system wired up.
  • Added Disclaimer link and associated modal dialog (#11).
  • Fleshed out initial draft of QueryLayer widget. (#10).
    • Toggle features on the map with checkbox.
    • Shows layer description and links to meta data page.
  • Grid/Identify window toggle. Not exposed in the UI yet (#9).
  • Added simple user management including request access, sign in, change/reset password. No custom geographic extent or time frame restrictions yet (#8).
  • Added reference layers popup (#6).
  • Added reference layers toggle controls and base map selector (#7).
